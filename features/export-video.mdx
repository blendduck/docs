# Export Video

This guide explains how to export a video from an existing project using the BlendDuck SDK.

## Usage

To export a video, you'll use the `projects.exportVideo()` method of the BlendDuck client. Here's a step-by-step guide:

1. Initialize the BlendDuck client
2. Call the `projects.exportVideo()` method with the project ID and export options
3. Monitor the export progress
4. Retrieve the exported video URL

Here's an example:

```typescript
import BlendDuck, { ExportOptions, ExportStatus } from "@blendduck/node-sdk";

async function exportVideo(projectId: string) {
  // Initialize the BlendDuck client
  const client = new BlendDuck({
    apiKey: process.env.BLENDDUCK_API_KEY
  });

  // Define export options
  const exportOptions: ExportOptions = {
    resolution: "1080p",
    format: "mp4",
    quality: "high",
    fps: 30
  };

  try {
    // Start the export process
    const exportId = await client.projects.exportVideo(projectId, exportOptions);

    console.log(`Export process started. Export ID: ${exportId}`);

    // Monitor the export progress
    let status: ExportStatus;
    do {
      await new Promise(resolve => setTimeout(resolve, 5000)); // Wait for 5 seconds
      status = await client.projects.getExportStatus(exportId);
      console.log(`Export progress: ${status.progress}%`);
    } while (status.status === 'processing');

    if (status.status === 'completed') {
      console.log(`Export completed successfully!`);
      console.log(`Download your video at: ${status.downloadUrl}`);
    } else {
      console.error(`Export failed: ${status.error}`);
    }
  } catch (error) {
    console.error("Error exporting video:", error);
  }
}

// Usage
exportVideo("your-project-id-here");
```

## Export Options

The `ExportOptions` object allows you to customize your video export:

- `resolution`: The video resolution (e.g., "720p", "1080p", "4K")
- `format`: The output format (e.g., "mp4", "webm")
- `quality`: The export quality (e.g., "low", "medium", "high")
- `fps`: The frames per second for the exported video

## Export Status

The `getExportStatus()` method returns an `ExportStatus` object with the following properties:

- `status`: The current status of the export ("processing", "completed", or "failed")
- `progress`: The export progress as a percentage (0-100)
- `downloadUrl`: The URL to download the exported video (available when status is "completed")
- `error`: Error message if the export failed

## Error Handling

If there's an issue with starting the export or with the API request, the method will throw an error. It's good practice to wrap the call in a try-catch block to handle any potential errors gracefully.

## Best Practices

1. **Quality vs File Size**: Choose the appropriate resolution, quality, and format based on your needs. Higher quality exports will result in larger file sizes.
2. **Progress Monitoring**: For long videos, implement a user-friendly way to display export progress, such as a progress bar or periodic notifications.
3. **Timeout Handling**: Implement a timeout mechanism for very long exports to prevent indefinite waiting.
4. **Retry Logic**: Consider implementing retry logic for failed exports, as temporary issues might resolve themselves.
5. **Parallel Exports**: If exporting multiple videos, consider implementing a queue system to manage parallel exports efficiently.
